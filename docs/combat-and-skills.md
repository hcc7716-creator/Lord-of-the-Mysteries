# 战斗与技能系统设计文档

> 文档用途：定义游戏中的战斗定位、技能表达方式、状态效果、技能数据字段和 MVP 技能样例。  
> 适用范围：区域开放世界剧情 RPG 的战斗系统、调查系统、技能 UI、AI 行为、状态机、任务事件和后续 Codex 数据开发。  
> 数据联动：本系统应与 `data/abilities.json`、`data/sequences.json`、`data/pathways.json` 和 `docs/potion-advancement-system.md` 联动。

## 1. 系统目标

本项目的技能系统不能只写“某个序列有什么能力”，而要把能力拆成玩家可以理解、可以使用、可以升级、可以失败、可以被 AI 调用的游戏技能。

技能文档需要回答以下问题：

1. 玩家什么时候能用这个技能。
2. 技能消耗什么资源。
3. 技能是否有冷却。
4. 技能命中或成功后改变什么游戏状态。
5. 技能失败、误用或超阶使用会带来什么风险。
6. 技能在 UI、动画、音效和 AI 行为中如何表现。
7. 技能如何与理智值、污染值、灵性值、锚点值、消化度和神秘知识承受度联动。

因此，“灵摆占卜”不只是占卜家能力，而是一个调查技能；“纸人替身”不只是魔术师能力，而是一个防御和逃生技能；“灵视”不只是设定描述，而是一个可开关、会消耗灵性、会暴露污染风险的探索技能。

## 2. 战斗定位

游戏战斗不应做成纯数值砍杀，而应强调“准备、判断、代价和异常状态”。

核心体验：

- 战斗前通过调查、占卜、灵视、社交和材料准备降低风险。
- 战斗中通过位移、防御、控制、仪式、召唤、逃生和状态管理取得优势。
- 战斗后根据污染、理智、灵性和锚点状态结算后遗症。
- 玩家可以赢得战斗，但不一定能无代价地赢。

MVP 建议采用“即时行动 + 技能冷却 + 少量战术暂停”的结构：

| 模块 | 说明 |
| --- | --- |
| 普通行动 | 移动、互动、普通攻击、闪避、格挡、使用道具 |
| 主动技能 | 消耗灵性、材料、弹药、道具或特殊资源 |
| 被动技能 | 持续改变判定、抗性、感知、闪避或风险 |
| 仪式技能 | 需要地点、材料、准备时间或任务条件 |
| 调查技能 | 用于案件、线索、物品、NPC、区域异常 |
| 逃生技能 | 避免致命伤害、脱离锁定、替身、位移、隐匿 |
| 风险结算 | 通过污染、理智、灵性透支、锚点动摇形成代价 |

## 3. 技能设计原则

### 3.1 每个技能必须是可执行动作

错误写法：

| 写法 | 问题 |
| --- | --- |
| 占卜家能占卜 | 太宽泛，无法实现，也无法平衡 |
| 魔术师会纸人替身 | 没有消耗、触发条件和失败后果 |
| 太阳途径能净化 | 不知道净化什么、净化多少、冷却多久 |

推荐写法：

| 字段 | 示例 |
| --- | --- |
| 技能名 | 灵摆占卜 |
| 类型 | 调查技能 |
| 消耗 | 5 灵性 |
| 冷却 | 无 |
| 效果 | 判断目标物品是否与当前案件有关 |
| 失败风险 | 若目标涉及高位存在，污染值 +10 |
| AI 实现备注 | 对案件物品调用 `case_link_score`，结果按可信度显示 |

### 3.2 技能需要有明确玩法分类

| 技能类型 | 用途 | 常见途径 |
| --- | --- | --- |
| 调查 | 寻找线索、判断关联、读取痕迹 | 愚者、白塔、隐者、门 |
| 战斗 | 造成伤害、压制敌人、打断施法 | 暴君、红祭司、太阳、黄昏巨人 |
| 防御 | 护盾、格挡、减伤、替身 | 太阳、黄昏巨人、愚者、审判者 |
| 逃生 | 闪避、位移、脱离、隐匿 | 愚者、门、错误、魔女 |
| 控制 | 束缚、眩晕、沉默、减速、恐惧 | 黑夜、被缚者、审判者、空想家 |
| 仪式 | 净化、封印、献祭、召唤、占卜 | 愚者、太阳、隐者、白塔、母亲 |
| 辅助 | 治疗、净化、增益、状态稳定 | 太阳、母亲、月亮、空想家 |
| 召唤 | 秘偶、灵体、构装、仆从 | 愚者、死神、完美者、深渊 |
| 被动 | 感知、抗性、危险预警、身份伪装 | 多数途径 |

### 3.3 技能必须能失败或产生副作用

诡秘风格的技能不是“按下按钮必定收益”。失败和副作用是系统核心。

| 风险来源 | 可能后果 |
| --- | --- |
| 目标位格过高 | 污染值上升、占卜失败、获得错误线索 |
| 灵性不足 | 技能中断、理智值下降、进入灵性透支 |
| 连续使用 | 冷却延长、命运干扰值上升、AI 警觉 |
| 材料替代 | 效果降低、污染上升、状态异常 |
| 场地不稳定 | 仪式失败、召来异常、封印松动 |
| 锚点过低 | 身份技能失控、伪装难以解除、自我认知下降 |

## 4. 技能与状态效果表

### 4.1 技能字段

此表是技能数据的正式字段建议。中文字段用于策划和文档显示，英文字段用于后续代码、JSON 和数据库。

| 中文字段 | 英文字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 技能 ID | `skill_id` | string | 小写英文加下划线，例如 `skill_seer_pendulum_divination` |
| 技能名 | `skill_name_cn` | string | 玩家看到的中文名 |
| 英文名 | `skill_name_en` | string | 数据库、代码和英文 UI 使用 |
| 所属途径 | `pathway_id` | string | 例如 `fool`、`door`、`sun` |
| 解锁序列 | `unlock_sequence` | number | 9 到 0 |
| 技能类型 | `skill_type` | string | investigation、combat、defense、escape、control、ritual、support、summon、passive |
| 消耗 | `cost` | object | 灵性、材料、道具、生命、理智、特殊资源 |
| 冷却 | `cooldown` | number/string | 秒数或 `none` |
| 效果 | `effect` | string | 技能在游戏里的实际效果 |
| 升级效果 | `upgrade_effect` | string | 高序列、熟练度或消化度提升后的增强 |
| 副作用 | `side_effect` | string | 失败、连续使用、目标过强或污染环境下的风险 |
| 视觉表现 | `visual` | string | UI、动画、特效、镜头或音效表现 |
| AI 实现备注 | `ai_implementation_note` | string | NPC AI、敌人 AI、任务系统或判定逻辑备注 |
| 状态标签 | `gameplay_tags` | string[] | 用于检索和系统联动 |
| 资料来源 | `source_note` | string[] | 原著、Wiki、官方补充或游戏化改编 |
| 正典状态 | `canon_status` | string | 参考途径数据结构文档中的 canon_status |

### 4.2 状态效果字段

状态效果用于记录燃烧、污染、理智动摇、眩晕、束缚、净化、标记、命运干扰等持续效果。

| 中文字段 | 英文字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 状态 ID | `status_id` | string | 例如 `status_polluted`、`status_fate_interference` |
| 状态名 | `status_name_cn` | string | 玩家看到的状态名 |
| 状态类型 | `status_type` | string | buff、debuff、dot、control、risk、hidden、environment |
| 触发来源 | `source` | string | 技能、道具、区域、封印物、剧情或仪式 |
| 持续时间 | `duration` | number/string | 秒数、回合数、任务阶段或 `permanent_until_cleansed` |
| 叠加规则 | `stack_rule` | string | 不叠加、刷新时间、叠层、取最高、转化为高阶状态 |
| 数值效果 | `mechanical_effect` | string | 对属性、判定、伤害、抗性、AI 行为的影响 |
| 解除方式 | `remove_method` | string | 休息、净化、道具、任务、仪式、等待 |
| UI 表现 | `ui_hint` | string | 图标、颜色、闪烁、噪点、文字提示 |
| AI 实现备注 | `ai_implementation_note` | string | 敌我 AI 如何识别、规避或利用该状态 |

### 4.3 可选实现字段

当技能进入原型开发阶段，建议在上表基础上补充以下字段，用于关卡、输入、AI 和测试：

| 中文字段 | 英文字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 使用场景 | `usage_context` | string[] | combat、exploration、dialogue、ritual、quest_node |
| 玩家目标 | `player_intent` | string | 玩家为什么会选择使用该技能 |
| 输入方式 | `input_method` | string | 按键、长按、瞄准、自动触发、交互点触发 |
| 目标类型 | `target_type` | string | self、enemy、ally、area、item、clue、ritual_node |
| 释放条件 | `cast_condition` | string | 需要视线、距离、火源、材料、地点、状态等 |
| 释放时间 | `cast_time` | number/string | 瞬发、读条、持续、战前准备 |
| 边界情况 | `edge_cases` | string[] | 无目标、目标免疫、资源不足、落点非法、同帧多次触发 |
| 调参变量 | `tuning_variables` | string[] | 范围、持续、消耗、冷却、成功率、误判率、风险值 |
| 测试标准 | `test_standard` | string | 判断技能是否好用、过强、过弱或不可读 |

## 5. 核心资源与风险

### 5.1 技能消耗类型

| 消耗类型 | 示例 | 设计用途 |
| --- | --- | --- |
| 灵性 | 5 灵性、15 灵性 | 最常见技能资源 |
| 材料 | 1 纸人、1 银粉、1 圣水 | 强化准备感和背包系统 |
| 理智 | 扣除 5 理智 | 高风险知识、梦境或禁忌技能 |
| 污染 | 污染值 +10 | 强行使用污染力量的代价 |
| 锚点 | 锚点值 -5 | 伪装、身份替换、精神系技能代价 |
| 生命 | 当前生命值 10% | 血肉、献祭、深渊或月亮系技能 |
| 冷却占用 | 30 秒冷却 | 限制强力技能频率 |

### 5.2 失败风险类型

| 风险类型 | 触发条件 | 常见后果 |
| --- | --- | --- |
| 占卜反噬 | 目标位格过高或被遮蔽 | 污染值上升、获得错误提示 |
| 灵性透支 | 当前灵性不足仍强行使用 | 理智值下降、技能效果变弱 |
| 命运干扰 | 连续替身、强行闪避死亡 | 后续随机事件更危险 |
| 锚点动摇 | 长期伪装、精神侵入 | 身份技能失控、对话选项异常 |
| 仪式失败 | 材料、地点、时间或步骤错误 | 召来异常、污染区域、封印松动 |
| AI 警觉 | 对同一目标反复使用调查技能 | NPC 敌意上升、敌人改变巡逻 |

## 6. 技能表样例

以下样例不是完整 220 序列技能库，而是给后续填表使用的格式模板。后续可从 `data/abilities.json` 中继续扩展。

| 技能名 | 所属途径 | 解锁序列 | 技能类型 | 消耗 | 冷却 | 效果 | 升级效果 | 副作用 | 视觉表现 | AI 实现备注 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 灵摆占卜 | 愚者 | 序列 9 占卜家 | 调查 | 5 灵性 | 无 | 判断目标物品、地点或人物是否与当前案件有关，返回“有关 / 无关 / 被遮蔽 / 危险” | 消化度提升后显示可信度和方向提示 | 若目标涉及高位存在或污染源，污染值 +10，可能生成误导结果 | 灵摆在 UI 中左右摆动，答案以短句和符号出现 | 调用案件系统的 `case_link_score`，若目标有 `high_entity_tag` 则追加污染检定 |
| 纸笔占卜 | 愚者 | 序列 9 占卜家 | 调查 | 2 灵性 + 1 纸张 | 10 秒 | 在案件笔记中生成一个关键词提示 | 可自动标记矛盾线索或隐藏关联 | 高污染时可能生成错误关键词 | 纸面浮现墨迹，关键词逐渐稳定 | 从任务线索池抽取低剧透提示，避免直接给最终答案 |
| 灵视 | 愚者 / 通用神秘学 | 序列 9 起 | 调查 / 开关 | 每秒 1 灵性 | 关闭后 5 秒 | 高亮灵体痕迹、污染残留、仪式节点和异常气场 | 高序列可显示污染类型、危险等级和残留时间 | 长时间开启会获得“灵视疲劳”，污染环境中理智值下降 | 屏幕边缘出现冷色光晕，异常物体出现轮廓 | AI 只暴露带 `spiritual_trace` 的对象，避免全图扫描 |
| 梦境占卜 | 愚者 | 序列 9 占卜家 | 调查 / 仪式 | 8 灵性 + 安全地点 | 60 秒 | 进入短梦境片段，获得象征线索或危险预兆 | 可延长片段，解锁更多关键词 | 梦境被污染时可能触发梦境战斗或理智值 -10 | 画面进入梦境滤镜，关键词漂浮出现 | 从剧情节点读取 `dream_hint_pool`，按污染值混入错误象征 |
| 危险预感 | 愚者 | 序列 8 小丑 | 被动 / 生存 | 无 | 无 | 致命攻击或陷阱触发前给出短暂方向预警 | 预警窗口随消化度增加 | 高污染时会误报或漏报 | 屏幕边缘出现短促闪光 | 敌人攻击前 0.4-0.8 秒触发提示，MVP 不做复杂预知 |
| 滑稽闪避 | 愚者 | 序列 8 小丑 | 逃生 | 4 灵性 | 10 秒 | 用夸张动作闪避一次近距离攻击 | 可在完美时机触发反击窗口 | 连续使用增加体力消耗，失败时硬直更长 | 角色翻滚、滑步或诡异侧身 | AI 可把它当作短距离闪避，优先脱离 AOE |
| 纸人替身 | 愚者 | 序列 7 魔术师 | 防御 / 逃生 | 1 纸人 + 15 灵性 | 90 秒 | 抵挡一次致命伤害，并在短距离外生成本体 | 纸人可短暂吸引敌人仇恨 | 连续使用会提升命运干扰值，后续遭遇更容易出现意外 | 本体化成纸屑，纸人被攻击后燃尽 | 触发条件为 `incoming_damage >= lethal_threshold`，同时写入 `fate_interference` |
| 火焰跳跃 | 愚者 | 序列 7 魔术师 | 位移 / 逃生 | 12 灵性 + 火源 | 20 秒 | 在可见火源之间短距离瞬移 | 高序列可携带一名友方或跨越障碍 | 火源被污染时可能偏移落点 | 火焰拉成长线，角色在火光中消失并出现 | 寻找半径内 `fire_anchor`，无火源时禁用 |
| 空气弹 | 暴君 | 序列 8 暴怒之民 | 战斗 | 6 灵性 | 3 秒 | 发射压缩空气造成冲击伤害并打断轻型敌人 | 可蓄力增加击退距离 | 水下、风暴区或狭窄空间会改变弹道 | 空气扭曲成透明弹道 | 敌人 AI 可尝试侧向闪避，命中后打断 `casting` |
| 雷击 | 暴君 | 序列 7 航海家 | 战斗 / 控制 | 14 灵性 | 18 秒 | 对目标区域造成雷电伤害，并短暂麻痹敌人 | 雨天或潮湿区域伤害提高 | 误击水面可能伤及友方或自己 | 天空或武器引下蓝白闪电 | 对带 `wet` 状态目标追加麻痹时间 |
| 圣光净化 | 太阳 | 序列 8 祈光人 | 辅助 / 净化 | 10 灵性 | 25 秒 | 清除一个轻度污染或诅咒状态 | 可同时恢复少量理智值 | 对深层污染效果降低，强行净化可能反噬 | 金色光束照亮目标，污染阴影蒸发 | 优先选择 `cleansable_level <= 1` 的状态 |
| 光刃 | 太阳 | 序列 7 太阳神官 | 战斗 | 8 灵性 | 6 秒 | 释放光属性斩击，对亡灵、阴影和污染体增伤 | 可升级为穿透或范围光刃 | 在隐蔽任务中使用会暴露位置 | 金色刃光横扫，命中后留下灼痕 | AI 对 `undead`、`shadow`、`polluted` 标签提高权重 |
| 入梦安抚 | 空想家 | 序列 7 心理医生 | 辅助 / 控制 | 12 灵性 | 45 秒 | 安抚目标情绪，降低敌意或解除轻度疯狂 | 可在对话中开启特殊选项 | 对高污染目标失败会导致施术者理智值下降 | 目标周围出现柔和梦境波纹 | NPC AI 将 `hostility` 降低一档，Boss 只吃短暂控制 |
| 心理暗示 | 空想家 | 序列 8 读心者 | 控制 / 社交 | 8 灵性 | 30 秒 | 在短时间内提高说服、威慑或潜入成功率 | 可植入简单行动倾向 | 失败会让目标警觉并记录玩家异常 | 目标眼神短暂失焦，字幕出现低语感 | 对 `willpower` 高的 NPC 需要额外检定 |
| 门径闪现 | 门 | 序列 7 占星人 | 位移 / 逃生 | 15 灵性 | 25 秒 | 在视线范围内短距离穿越障碍 | 可标记安全落点并减少冷却 | 空间污染区域可能产生错位伤害 | 空间裂开细缝，角色折叠位移 | 需要 NavMesh 可达落点，禁止穿越剧情封锁 |
| 开锁术 | 门 | 序列 9 学徒 | 调查 / 潜入 | 4 灵性 | 5 秒 | 打开普通机械锁或弱神秘封印 | 可识别锁的来源和陷阱 | 高等级封印失败会触发警报 | 锁孔周围出现星光纹路 | 与锁对象 `lock_level` 比较，失败可触发 `alarm_event` |

| 黑暗帷幕 | 黑夜 | 序列 7 梦魇 | 控制 / 潜入 | 10 灵性 | 30 秒 | 降低区域内敌人视野，并提高玩家潜行成功率 | 可附加短暂恐惧或睡意 | 强光区域效果下降 | 区域被深蓝黑暗覆盖，声音变闷 | 修改敌人感知半径和视线角度 |
| 死亡标记 | 死神 | 序列 7 通灵者 | 战斗 / 诅咒 | 10 灵性 | 20 秒 | 标记一个敌人，使其受到亡灵或灵体伤害提高 | 可在目标死亡后生成短暂灵体回响 | 连续标记会吸引附近亡灵 | 目标身上出现灰白烙印 | 给目标添加 `death_marked`，召唤物优先攻击 |
| 战争号令 | 红祭司 | 序列 7 纵火家 | 战斗 / 团队 | 14 灵性 | 60 秒 | 提高附近友方攻击欲望和冲锋速度 | 可追加燃烧抗性或士气免疫 | 理智低时可能使友方鲁莽进攻 | 红色火光沿地面扩散，队友武器发亮 | 友方 AI 进攻权重提高，防御权重降低 |
| 镜面误导 | 魔女 | 序列 7 女巫 | 控制 / 逃生 | 10 灵性 + 镜面 | 35 秒 | 制造镜像误导敌人锁定 | 可让镜像承受一次普通攻击 | 镜面被污染时可能出现敌对倒影 | 镜面裂出虚像，人物轨迹重叠 | 敌人 AI 目标选择加入 `decoy_target` |
| 血肉修补 | 母亲 / 月亮 | 序列 8 起 | 辅助 / 治疗 | 12 灵性 + 药材 | 30 秒 | 恢复目标生命并止血 | 可清除轻度创伤状态 | 污染环境下可能留下异常增生 | 肉芽和绿光快速缝合伤口 | 不应治疗污染性畸变，需调用净化技能 |

## 8. 技能升级规则

技能升级不建议只做“伤害 +10%”。更适合从以下方向扩展：

| 升级方向 | 说明 | 示例 |
| --- | --- | --- |
| 精度提升 | 结果更稳定，错误信息减少 | 灵摆占卜显示可信度 |
| 范围扩大 | 影响更多目标或更大区域 | 黑暗帷幕覆盖整条巷道 |
| 风险降低 | 污染、理智或命运干扰惩罚减少 | 纸人替身连续使用惩罚降低 |
| 条件放宽 | 不再需要特定材料或地点 | 火焰跳跃可使用微弱火星 |
| 连携增强 | 与其他技能或状态产生新效果 | 死亡标记让召唤灵体优先攻击 |
| UI 信息增强 | 给玩家更多可读信息 | 灵视显示污染类型 |

升级来源：

- 当前序列消化度达到阶段阈值。
- 晋升到更高序列。
- 完成技能专属任务。
- 获得导师教学或神秘学笔记。
- 使用材料强化技能媒介。

## 9. AI 实现规则

### 9.1 玩家技能 AI 备注

玩家技能中的 AI 实现备注主要服务于系统和任务：

- 调查技能需要说明读取哪个线索池。
- 位移技能需要说明落点是否合法。
- 防御技能需要说明触发条件。
- 控制技能需要说明哪些敌人免疫或减效。
- 仪式技能需要说明地点、时间、材料和剧情开关。
- 风险技能需要说明如何写入污染、理智、锚点或命运干扰。

### 9.2 敌人和 NPC 使用技能

敌人不应该无脑释放技能，而要根据状态和目标选择。

| AI 条件 | 行为 |
| --- | --- |
| 玩家灵性透支 | 优先使用精神攻击或压制技能 |
| 玩家被标记 | 追踪型敌人优先集火 |
| 玩家污染过高 | 神秘敌人更容易锁定玩家 |
| 玩家使用强光技能 | 阴影类敌人后退或寻找遮蔽 |
| 玩家连续纸人替身 | 敌人尝试范围攻击或封锁逃生点 |
| NPC 理智动摇 | 社交对话更容易失败或转入异常台词 |

## 11. MVP 实现范围

第一版建议先做少量高辨识度技能，不急着把 22 条途径全部做成完整战斗职业。

优先实现：

- 愚者途径：灵摆占卜、灵视、危险预感、纸人替身、火焰跳跃。
- 太阳途径：圣光净化、光刃。
- 暴君途径：空气弹、雷击。
- 黑夜途径：黑暗帷幕、入梦控制。
- 空想家途径：入梦安抚、心理暗示。
- 门途径：开锁术、门径闪现。
- 通用状态：污染、理智动摇、灵性透支、命运干扰、被标记、净化、燃烧、麻痹。

暂时不做：

- 所有序列的完整技能树。
- 大规模多人队伍战斗。
- 复杂卡牌式技能构筑。
- 每个技能的完整动画资源。
- 高序列神级能力的完全开放使用。
- 能直接跳过主线推理的万能占卜。

## 12. MVP 验收标准

MVP 阶段不要求技能数量多，但每个技能都必须证明一个明确玩法方向。

| 技能 | 验证目标 | 通过标准 |
| --- | --- | --- |
| 灵摆占卜 | 信息技能能否影响玩家路线选择 | 玩家能根据提示选择调查方向，但不能直接跳过案件 |
| 灵视 | 隐藏信息是否能形成探索和战斗决策 | 玩家能发现灵体、污染痕迹或仪式节点，并理解开启成本 |
| 危险预感 | 被动预警是否能改善生存体验 | 玩家能看懂危险方向，但不能无脑免疫伤害 |
| 纸人替身 | 高风险战斗是否有可预谋保命手段 | 玩家愿意提前准备纸人，连续使用会感到代价真实存在 |
| 火焰跳跃 | 位移是否让战斗产生空间博弈 | 玩家会主动寻找火源、制造火源或规避无落点区域 |
| 圣光净化 | 净化是否能处理污染与异常状态 | 玩家能区分轻度污染可净化、深层污染需任务处理 |
| 入梦安抚 | 非击杀控制是否有价值 | 玩家能用它打断、降敌意或救 NPC，但不能长期控制 Boss |
| 雷击 | 环境联动是否能提高战斗策略 | 潮湿、水面、金属目标能产生可读的战术变化 |

验收时需要同时检查：

- 玩家能否在 UI 中看懂消耗、冷却和风险。
- 技能失败是否产生新的局面，而不是单纯无事发生。
- AI 是否会对玩家技能做出基本反应。
- 状态效果是否能被复用，而不是每个技能单独写死。
- 高收益技能是否有污染、理智、材料、冷却或命运干扰代价。

## 13. 总结

战斗与技能系统的核心不是把设定能力逐字搬进游戏，而是把能力变成有消耗、有冷却、有失败风险、有状态反馈、有 AI 行为备注的可执行技能。

技能表是后续开发的桥梁：

- 对玩家，它决定技能 UI 和战斗体验。
- 对策划，它决定数值、冷却、风险和升级路线。
- 对任务系统，它决定调查、线索、仪式和剧情开关。
- 对 AI，它决定敌人和 NPC 如何识别、使用、规避这些能力。
- 对数据开发，它决定 `data/abilities.json` 后续如何从能力资料升级为可运行技能库。
