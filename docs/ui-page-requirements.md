# UI 页面需求文档

> 文档用途：定义游戏 MVP 阶段主要 UI 页面的用途、按钮、显示数据、玩家操作和页面跳转关系。  
> 适用范围：主菜单、角色成长、途径与魔药、背包、任务、地图、调查、秘密会议、封印物管理、污染与理智状态。  
> 关联文档：`docs/potion-advancement-system.md`、`docs/combat-and-skills.md`、`docs/quests-and-story-volume-1.md`、`docs/item-material-artifact-database.md`、`docs/npcs-and-factions.md`。  
> 关联数据：`data/items.json`、`data/pathways.json`、`data/sequences.json`、`data/materials.json`、`data/abilities.json`、`data/rituals.json`、`data/sealed_artifacts.json`。

## 1. 设计目标

UI 的目标不是把所有设定堆给玩家，而是让玩家能快速判断：

1. 我现在是谁。
2. 我正在调查什么。
3. 我能去哪里。
4. 我缺什么材料。
5. 我是否适合晋升。
6. 我是否接近污染或失控。
7. 我可以向哪个势力求助或交易。

第一版 UI 以可开发、可测试为主，先保证页面结构、数据来源和跳转关系清晰。视觉风格可以后续独立做。

## 2. 全局导航规则

建议在游戏内提供一个统一暂停/菜单入口，打开后能进入以下页面：

| 页面 | 入口 |
| --- | --- |
| 角色面板 | 暂停菜单、快捷键、角色头像 |
| 途径面板 | 角色面板、魔药调配界面 |
| 魔药调配界面 | 安全屋、炼金台、途径面板 |
| 材料背包 | 暂停菜单、魔药调配界面、商店 |
| 任务日志 | 暂停菜单、地图界面、调查笔记 |
| 地图界面 | 暂停菜单、任务日志 |
| 调查笔记 | 暂停菜单、任务日志、案件现场 |
| 塔罗会/秘密会议界面 | 安全屋、剧情节点、主菜单继续游戏后特殊入口 |
| 封印物管理界面 | 安全屋、守灯人组织据点、背包封印物详情 |
| 污染与理智状态界面 | 角色面板、污染警告提示、晋升前检查 |

通用按钮：

| 按钮 | 功能 |
| --- | --- |
| 返回 | 回到上一级页面 |
| 关闭 | 关闭 UI 回到游戏 |
| 帮助 | 打开当前页面简短说明 |
| 标记 | 将任务、材料、地图点或线索设为追踪 |
| 详情 | 打开选中对象详情 |

跳转约定：

- 页面跳转需要记录来源页面，例如从任务日志进入地图后，地图的“返回”应回到该任务详情，而不是固定返回暂停菜单。
- 重要操作必须二次确认，包括服用魔药、销毁材料、转移封印物、推进关键推理、进入秘密会议、执行污染压制方案。
- 筛选、排序、详情抽屉、风险预览和追踪选择器应做成通用组件，在背包、任务、地图、调查笔记、封印物页面复用。

## 3. 通用弹窗与子页面

| 组件 | 用途 | 调用页面 |
| --- | --- | --- |
| 确认弹窗 | 服用魔药、销毁材料、转移封印物、提交关键推理、进入高污染区前二次确认 | 魔药调配、材料背包、封印物管理、调查笔记、地图 |
| 详情抽屉 | 快速查看能力、材料、线索、任务、NPC、封印物，不打断当前页面上下文 | 全部资料型页面 |
| 追踪选择器 | 将任务、材料、线索、地图点写入 HUD 追踪目标 | 任务日志、地图、材料背包、调查笔记、魔药调配 |
| 风险预览弹窗 | 显示预计污染变化、理智损耗、失败后果、是否可撤销 | 魔药调配、封印物管理、污染与理智、秘密会议 |
| 筛选排序组件 | 按类型、稀有度、途径、任务状态、区域、风险等级筛选 | 背包、任务、地图、调查笔记、封印物管理 |
| 数量选择弹窗 | 拆分、丢弃、交易、投入材料时选择数量 | 材料背包、商店、魔药调配 |

## 4. 主菜单

### 页面用途

主菜单用于开始游戏、继续存档、进入设置、查看制作信息，以及在后续版本进入资料库或模组/开发工具入口。

### 包含按钮

| 按钮 | 说明 |
| --- | --- |
| 继续游戏 | 读取最近存档 |
| 新游戏 | 创建新角色并进入第一卷开场 |
| 读取存档 | 打开存档列表 |
| 设置 | 打开音频、画面、控制、语言设置 |
| 资料库 | 查看已解锁世界观、怪物、封印物和途径资料 |
| 退出游戏 | 关闭游戏 |

### 显示数据

| 数据 | 来源 |
| --- | --- |
| 最近存档名称 | 存档系统 |
| 当前章节 | 任务系统 |
| 玩家当前序列 | 角色存档 |
| 游戏版本号 | 构建配置 |

### 玩家可以做什么操作

- 继续最近存档。
- 新建角色并进入开场。
- 读取指定存档。
- 调整设置。
- 查看已解锁资料。
- 退出游戏。

### 点击后跳转哪里

| 点击 | 跳转 |
| --- | --- |
| 继续游戏 | 游戏主场景 |
| 新游戏 | 角色创建或第一卷开场 |
| 读取存档 | 存档列表 |
| 设置 | 设置界面 |
| 资料库 | Codex / 资料库界面 |
| 退出游戏 | 系统退出确认 |

## 17. 总结

UI 页面不是孤立菜单，而是把项目核心系统串起来的操作层：

- 角色面板回答“我是谁”。
- 途径面板回答“我要成长到哪里”。
- 魔药调配回答“我缺什么，风险多高”。
- 材料背包回答“我有什么，能怎么用”。
- 任务日志回答“我现在该做什么”。
- 地图回答“我要去哪里”。
- 调查笔记回答“我知道了什么”。
- 秘密会议回答“我如何远程经营关系和情报”。
- 封印物管理回答“我如何使用高风险道具”。
- 污染与理智状态回答“我还能承受多少代价”。
